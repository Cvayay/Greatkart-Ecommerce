# Contributing

Thanks for your interest in contributing to Greatkart.

Guidelines
- Fork the repository and create a feature branch: `git checkout -b my-feature`
- Keep commits small and focused. Write clear commit messages.
- Run linters/tests before opening a pull request.

Development workflow
1. Fork the repo on GitHub.
2. Clone your fork and set `origin` to your fork.
3. Rebase or merge upstream `main` frequently to keep branches up to date.

Reporting issues
- Open issues for bugs, feature requests, or documentation improvements.

Code style
- Follow PEP8 for Python code. Use black/flake8 as you prefer.
